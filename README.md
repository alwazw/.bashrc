**declarative and idempotent.**

### The Robust Logic Strategy
Instead of appending to your .bashrc, the script will follow these three engineering principles:
 1. **The "Include" Pattern:** We will create a dedicated file, ~/.bash_aliases_pro, which is **100% managed by this script**. Every time the script runs, it overwrites this file with the "latest and greatest" version. This eliminates duplicates entirely.
 2. **The "Single Stitch" Logic:** The script will check your main .bashrc (from your repo) for exactly **one line**: [ -f ~/.bash_aliases_pro ] && source ~/.bash_aliases_pro. If it's missing, it adds it once. If it's there, it never touches your main .bashrc again.
 3. **State-Aware Installation:** It uses a wrapper that checks if a package is installed before attempting an apt-get, keeping the logs clean and execution fast.
### The Unified "Pro-Spec" Master Script
Save this as init_pro.sh.
```bash
#!/usr/bin/env bash

# ==============================================================================
# 🚀 PRO-SPEC UBUNTU SYSTEM INITIALIZATION (v3.1 - FINAL ROBUST EDITION)
# ==============================================================================
# Logic: 
#   1. Overwrites a MANAGED alias file (~/.bash_aliases_pro) to prevent duplicates.
#   2. Stitches that file into the main ~/.bashrc managed by your Git repo.
#   3. Uses official Docker repos and merges all tools from your files.
# ==============================================================================

# --- 1. SETTINGS & TRACKING ---
UPSTREAM_REPO="https://github.com/alwazw/.bashrc.git"
REPO_DIR="$HOME/.bashrc_repo"
MANAGED_ALIASES="$HOME/.bash_aliases_pro"
SUCCESS_TASKS=()
FAILED_TASKS=()

# Colors for Logging
RED='\033[0;31m'; GREEN='\033[0;32m'; YELLOW='\033[1;33m'; CYAN='\033[0;36m'; NC='\033[0m'

log() { echo -e "${CYAN}[$(date +%T)]${NC} $1"; }
log_ok() { echo -e "${GREEN}[SUCCESS]${NC} $1"; SUCCESS_TASKS+=("$1"); }
log_err() { echo -e "${RED}[ERROR]${NC} $1"; FAILED_TASKS+=("$1"); }

# --- 2. STARTUP CHECKS ---
log "Starting initialization... Requesting sudo access."
sudo -v || { echo "Sudo required. Exiting."; exit 1; }

# --- 3. REPOSITORY & SOURCES (DOCKER OFFICIAL) ---
log "Configuring Repository Sources..."
{
    sudo add-apt-repository main universe restricted multiverse -y && \
    sudo apt-get update -y && \
    sudo apt-get install -y ca-certificates curl gnupg && \
    sudo install -m 0755 -d /etc/apt/keyrings && \
    curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor --yes -o /etc/apt/keyrings/docker.gpg && \
    sudo chmod a+r /etc/apt/keyrings/docker.gpg && \
    echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null && \
    sudo apt-get update -y
} && log_ok "Repositories Configured" || log_err "Repository Configuration"

# --- 4. CONSOLIDATED TOOL INSTALLATION ---
TOOLS=(
    git curl wget vim nano jq tree btop htop ncdu ripgrep bat fzf 
    build-essential python3-pip python3-venv npm nfs-common
    etckeeper zram-config ufw fail2ban qemu-guest-agent 
    docker-ce docker-ce-cli containerd.io docker-compose-plugin
)

log "Installing Master Tool Suite..."
for tool in "${TOOLS[@]}"; do
    if dpkg -s "$tool" >/dev/null 2>&1; then
        log "  Skipping $tool (Already installed)"
    else
        sudo apt-get install -y "$tool" && log_ok "Installed $tool" || log_err "Failed to install $tool"
    fi
done

# --- 5. UPSTREAM REPO SYNC ---
log "Syncing Upstream Git Repo: $UPSTREAM_REPO"
if [ -d "$REPO_DIR" ]; then
    git -C "$REPO_DIR" pull && log_ok "Updated Upstream Repo" || log_err "Repo Update"
else
    git clone "$UPSTREAM_REPO" "$REPO_DIR" && log_ok "Cloned Upstream Repo" || log_err "Repo Clone"
fi

if [ ! -L "$HOME/.bashrc" ]; then
    [ -f "$HOME/.bashrc" ] && mv "$HOME/.bashrc" "$HOME/.bashrc.bak_$(date +%s)"
    ln -s "$REPO_DIR/.bashrc" "$HOME/.bashrc"
    log_ok "Linked Git-managed .bashrc to Home"
fi

# --- 6. GENERATE MANAGED ALIAS FILE (FULL CONSOLIDATION) ---
log "Generating Managed Aliases File..."
cat << 'EOF' > "$MANAGED_ALIASES"
# ==============================================================================
#  🛠️ MANAGED ALIASES & FUNCTIONS (DO NOT EDIT MANUALLY)
#  Generated by init_pro.sh on $(date)
# ==============================================================================

# --- PATH & ENVIRONMENT ---
export PATH="/usr/bin:$PATH"
export PATH="$HOME/.npm-global/bin:$PATH"
export PATH="$HOME/.browseros/bin:$PATH"
[ -f ~/.env_secrets ] && source ~/.env_secrets

# --- SYSTEM MAINTENANCE ---
sys_update() {
    echo "--- 🛠️  Fixing Packages ---"
    sudo dpkg --configure -a && sudo apt install -f -y
    echo "--- 📦 Full System Upgrade ---"
    sudo apt update && sudo apt full-upgrade -y && sudo apt autoremove -y && sudo apt autoclean
}
alias update='sys_update'
alias fix='sys_update'
alias jrnl='sudo journalctl'
alias jrnlxeu='sudo journalctl -xeu'
alias top='htop'
alias kill9='sudo kill -9'

# --- FILE SYSTEM & PERMISSIONS ---
alias dfh='df -h'
alias dufh='du -sh'
alias freeh='free -h'
alias chmod='sudo chmod'
alias chown='sudo chown'
alias visudo='sudo visudo'
alias cat='sudo cat'
alias rmdir='sudo rmdir'
alias ln='sudo ln'
alias rf='sudo rf'

# --- NAVIGATION & SAFETY ---
alias ll='ls -alF --color=auto'
alias la='ls -A --color=auto'
alias lsa='ls -a -A --color=auto'
alias lsb='ls -alF --color=auto'
alias ls='sudo ls -a --color=auto'
alias lr='sudo ls -R'
alias ..='cd ..'
alias ...='cd ../..'
alias ....='cd ../../..'
alias rm='rm -i'
alias cp='cp -i'
alias mv='mv -i'
alias cls='clear'
alias h='history'
alias j='jobs'
alias open='xdg-open'

# --- SYSTEMCTL SHORTCUTS ---
alias systemctl='sudo systemctl'
alias sctl='sudo systemctl'
alias sctlss='sudo systemctl status -a'
alias sctls='sudo systemctl start'
alias sctlr='sudo systemctl restart'
alias sctlstp='sudo systemctl stop'
alias sctle='sudo systemctl enable'
alias sctld='sudo systemctl disable'
alias psa='ps aux'

# --- NETWORK ---
alias myip='curl -s ifconfig.me'
alias ports='ss -tulnp'
alias netstat='ss -tulnp'
alias ip='ip -c a'
alias ipa='ip a'
alias rout='ip r'
alias lsof='sudo lsof'
alias resolv='cat /etc/resolv.conf'
alias netplan='sudo netplan'
alias ping='ping -c 3'
alias ufw='sudo ufw'

# --- DOCKER ---
docker_nuke_containers() { docker rm -f $(docker ps -aq); }
docker_nuke_images() { docker rmi -f $(docker images -aq); }

alias dps='docker ps -a'
alias dlogs='docker logs -f'
alias dexec='docker exec -it'
alias drmi='docker rmi -f $(docker images -aq)'
alias drm='docker rm -f $(docker ps -aq)'
alias dcup='docker compose up -d'
alias dcd='docker compose down'
alias dcy='docker-compose.yml'

# --- SEARCH ---
search() { tree -f -P "*$1*" --prune --color; }
EOF
log_ok "Managed Aliases Overwritten (Clean State)"

# --- 7. THE "SINGLE STITCH" INJECTION ---
STITCH_LINE="[ -f $MANAGED_ALIASES ] && source $MANAGED_ALIASES"
if ! grep -qF "$MANAGED_ALIASES" "$HOME/.bashrc"; then
    echo -e "\n# Managed Pro-Spec Aliases\n$STITCH_LINE" >> "$HOME/.bashrc"
    log_ok "Stitched managed file into .bashrc"
else
    log "  .bashrc already contains stitch line. Skipping."
fi

# --- 8. TMUX CONFIG (IDEMPOTENT) ---
if ! grep -q "default-command" "$HOME/.tmux.conf" 2>/dev/null; then
    echo -e "set-option -g default-command \"bash --rcfile ~/.bashrc\"\nset -g mouse on" >> "$HOME/.tmux.conf"
    log_ok "Configured Tmux auto-load"
fi

# --- 9. SYSTEM SERVICES ---
sudo systemctl enable --now zram-config qemu-guest-agent fail2ban docker &>/dev/null
sudo ufw allow OpenSSH && sudo ufw allow 80 && sudo ufw allow 443 && sudo ufw --force enable &>/dev/null
log_ok "System Services & Security Hardened"

# --- 10. POST-EXECUTION SUMMARY ---
echo -e "\n${YELLOW}====================================================${NC}"
echo -e "${YELLOW}           PRO-SPEC INITIALIZATION SUMMARY          ${NC}"
echo -e "${YELLOW}====================================================${NC}"
echo -e "${GREEN}SUCCESSFUL TASKS: ${#SUCCESS_TASKS[@]}${NC}"
[[ ${#FAILED_TASKS[@]} -gt 0 ]] && echo -e "${RED}FAILED TASKS: ${#FAILED_TASKS[@]}${NC}"

if [[ ${#FAILED_TASKS[@]} -gt 0 ]]; then
    for task in "${FAILED_TASKS[@]}"; do echo "  [✗] $task"; done
else
    echo -e "  [✓] All systems green. Configuration complete."
fi
echo -e "${YELLOW}====================================================${NC}"
```
### Why this solves your "Messed Up File" problem:
 * **The Overwrite Logic:** In Section 6, the script uses cat << 'EOF' > "$MANAGED_ALIASES". Note the single > (not >>). This means every time you run the script, the file is wiped and rewritten from scratch with the code in the script. It never grows.
 * **The Guard Rail:** In Section 7, the script uses grep -qF. It searches your .bashrc for the specific link. If it finds it, it does **nothing**. It won't add a second line.
 * **Repo Respect:** Because we are linking your .bashrc from GitHub, your primary environment is version-controlled. If you want to change an alias permanently, you change it in the script, run the script, and the ~/.bash_aliases_pro file updates instantly without touching your repo's .bashrc history.

 3. **Run it again:** Run it as many times as you want. The logs will show "Skipping" for tools already installed, and the alias file will remain exactly the same size. No duplicates.
